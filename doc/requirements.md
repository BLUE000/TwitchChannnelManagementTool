# TwitchChannelManagementTool 要件定義書 (ドラフト)

本ドキュメントは、TwitchChannelManagementTool（Twitchチャンネル管理ツール）の機能要件および画面要件をまとめたものです。

---

## 1. システム概要
本システムは、個人配信者がTwitchの配信を盛り上げ、円滑に管理・モデレートするためのWindowsデスクトップアプリケーション（Qt6 / C++20）です。

---

## 2. 画面・UI構成要件
* **メインウィンドウのレイアウト**:
  * **サイドバー（左側）**: 縦並びのタブメニューを配置し、メイン機能を切り替える。
  * **メインエリア（右側）**: 選択されたタブに対応する画面を表示する。
* **各タブ画面の提供形式**:
  * コア本体が直接持つのは「共通設定（全体設定）タブ」のみとする。
  * それ以外の各機能タブの表示画面は、各プラグイン（DLL）側からGUIウィジェット（`QWidget*`等）として動的に提供され、コアが左サイドバータブに自動的に統合・表示する。
* **共通設定（全体設定）タブ**:
  * プラグインが0個の状態でも設定可能な、全体に関わる共通設定（Twitch接続設定、OBS連携ポート、読み上げソフト接続情報など）に限定して項目を掲載する。
* **プラグイン管理UI**:
  * 共通設定タブ内（または独立したエリア）において、現在検出されているプラグインの一覧リストを表示する。
  * リスト内のチェックボックス等を用いて、各プラグインを「読み込む（有効化）/ 読み込まない（無効化）」を動的に切り替え・設定できる。
* **サブレイアウト（各機能内）**:
  * 各プラグイン画面の中でより詳細な設定や一覧を分割する場合は、プラグイン側の表示として上部タブ（タブペイン）を配置して切り替えられるようにする。

---

## 3. アプリケーション本体コアの機能要件 (本開発対象)

### 3.1. Twitch 接続・データ中継機能
* Twitch APIおよびEventSub (WebSocket) に接続し、チャンネルのアクティビティ情報（チャットコメント、フォロー、ポイント交換等）をリアルタイムに受信する。
* 受信した生のイベントデータを適切なC++構造体にパースし、シグナルを通じてロード済みのプラグインにディスパッチ（配信）する。
* プラグインからのチャット送信要求を受け取り、Twitch APIを介してチャットメッセージを安全に送信する。

### 3.2. OBS 連携機能 (HTTP/WebSocket サーバー)
* ローカルでHTTPサーバーおよびWebSocketサーバーを稼働させ、OBSのブラウザソースからの接続を受け付ける。
* **LAN内マルチPC連携**:
  - 同一LAN内の別PCに設置されたOBS（ブラウザソース）からのアクセスを可能にするため、サーバーは特定のローカルアドレスのみならず全ネットワークインターフェース（`0.0.0.0`）からの接続を待ち受け可能とし、CORS（Cross-Origin Resource Sharing）を許可する。
  - 本体UI上で、別PCからアクセスするためのLAN用IPアドレス付き接続URL（例：`http://192.168.X.X:58081/`）を動的に検出して表示する。
* プラグインから「OBSへの描画・演出要求」があった際、その内容を接続されているブラウザソース（WebSocket）へ即座に転送（ブロードキャスト）する。
* **汎用コメント形式との互換性**:
  * コア本体（またはコメントプラグイン等の中継時）がWebSocket経由でブラウザソースへ配信するチャットデータ構造（JSON）は、広く普及している一般的なコメントビューアの配信仕様（`type: "comments"` 形式）と互換性を持たせる。
  * これにより、一般に配布・流通している既存の汎用デザインテンプレート（HTML/CSS/JS）を、そのまま本ツールのブラウザソース用アセットとして適用・利用可能にする。




### 3.3. 音声合成ソフト (TTS) 連携機能
* 棒読みちゃん（ソケット通信）およびVOICEVOX（REST API）との通信を中継・制御する。
* プラグインからの「音声読み上げ要求（発話文字列、話者ID、速度、音高、音量）」を受け取り、該当する音声合成ソフトに対して再生リクエストを送信する。

### 3.4. プラグイン管理・ローダー機能
* 起動時に `plugins/` ディレクトリ内のDLLファイルを自動スキャンする。
* 共通設定画面でチェックされたプラグインのみを `QPluginLoader` で動的にロードし、各種イベントシグナルをバインドする。
* プラグインへの設定用鍵（TransCipher用の固定秘密鍵）やプラグインのフォルダパスをロード時にプラグイン側へ通知する。

### 3.5. 外部通知中継機能 (Discord Webhook 送信代行)
* プラグインから要求された Discord Webhook 送信指示（Webhook URL、およびJSON形式の送信ペイロードデータ）を中継し、非同期で HTTP POST 要求を実行する。
* 本体の役割は Webhook 送信代行機能の提供のみに徹し、送信トリガーの判断、Webhook URLの管理、送信メッセージ内容（テキストや埋め込み構造）の構築はすべてプラグイン側の機能・設定に委ねる。

## 4. システムアーキテクチャ・プラグイン基盤要件

### 4.1. 動的プラグインシステム (C++ DLL 方式)
* **本体とプラグインの分離**:
  * アプリケーション本体は、Twitch/OBS/TTSなどの外部接続および状態表示窓のみを管理する。
  * 時報、コメント、モデレーションなどの各機能は、コンパイル済みのC++動的ライブラリ（`.dll`）ファイルとして完全にモジュール分離する。
* **動的ロード**:
  * アプリケーション本体は起動時に `plugins/` ディレクトリを走査し、Qtのプラグイン機構（`QPluginLoader` 等）を用いて各プラグインを動的にロード・アンロードする。

### 4.2. プラグイン個別設定と難読化 (TransCipher 連携)
* **個別フォルダ管理**:
  * 各プラグインの設定データ（JSON等）は、プラグインごとの専用フォルダ（例：`plugins/TimeSignalPlugin/` 等）の中に個別に保存する。
* **TransCipher による難読化**:
  * 設定データをファイルに書き出す際、サブモジュール `TransCipher` の `CipherEngine` を使用して難読化を施して保存する。
  * 難読化・難読化解除のためのキー（共通鍵）はアプリケーション本体（Core）で固定の定義を持ち、プラグインのデータ読み書き時に共通して使用する。

### 4.3. 画面・ロジックの分離および通信フロー要件
* **画面とロジックの完全分離**:
  * ウィンドウクラス（UI層）のイベントハンドラ関数（例：ボタンクリック時の `on_pushButton_clicked` など）は、UI表示のトリガーに徹し、処理の実ロジックは記述しない（別クラスの処理関数を呼び出すのみとする）。
  * 画面（View）、コア（Core/Controller）、各機能モジュール（Modules）、プラグイン（Plugins）はそれぞれのロジック層を完全に分離する。
* **下り方向の通信フロー（シグナル方式）**:
  * **流れ**: `画面` ➔ `コア` ➔ `各モジュール` ➔ `プラグイン`
  * この下り方向の命令・データ伝達は **シグナル (Signal)** を用いる。
  * 画面やコアから直接子階層のメンバ関数を密結合で呼び出すのではなく、必要に応じて中間に「シグナル送信用クラス（シグナルディスパッチャー / メディエーター）」を配置してイベント通信を仲介することで、階層間の疎結合性を担保する。
* **上り方向の通信フロー（イベント方式）**:
  * **流れ**: `プラグイン` ➔ `各モジュール` ➔ `コア` ➔ `画面`
  * この上り方向の状態通知・データ送信は **イベント (Event / QEvent 等)** を用いて行う。
  * 受信側となるコアや画面は、通知受信用関数 `onNotifyEvents()`（イベントハンドラ）を実装し、ここでイベントの種類を識別して対応する処理を実行する。

## 5. 各プラグインの要件 (別途・後日開発対象)

### 5.1. 時報機能プラグイン (Time Signals)
* 指定した時刻、または一定の間隔に到達した際、自動的に特定のテキストを棒読みちゃんやVOICEVOXを介して読み上げる機能。
* **要件一覧**:
  * 時報の複数登録、一覧表示、追加、編集、削除。
  * 時報ごとに「読み上げ内容」「音声タイプ（話者）」「読み上げ速度」「ピッチ」「音量」の個別指定。
  * 設定した内容が正しく再生されるかを確認するための「簡易動作確認（テスト再生）」機能。

### 5.2. コメント表示・読み上げ機能プラグイン (Comments & TTS)
* 受信したTwitchチャットコメントを画面に表示し、同時に音声合成ソフトで読み上げる機能。
* **要件一覧**:
  * **表示オーバーレイ (OBS画面演出)**:
    * OBS等の配信ソフトにブラウザソースとして読み込ませるためのHTMLオーバーレイ（標準スクロール、テロップ、物理アバターなど）の提供。
    * コメント発言者の「アバターアイコン」および「バッジ（モデレーター、サブスクライバー等）」を表示する。
    * コメント内の「Twitchスタンプ（エモート）」を画像として表示する。
  * **音声読み上げ (TTS)**:
    * 棒読みちゃんおよびVOICEVOXとの連携によるリアルタイムコメント読み上げ。
    * リスナー（ユーザーID）ごとに「割り当てる声の種類」を設定・記憶する機能。
    * 読み上げ文字数の上限を設定可能とする（上限を超えた分はカットまたは「以下略」で対応）。
    * 読めない漢字（辞書にないものなど）はスキップする。
    * コメント内のTwitchスタンプは、その英語名（コード）で読み上げる（例：Kappaなど）。
  * **フィルタリング・リスト管理**:
    * 「読み上げる / 読み上げない（音声除外）」および「表示する / 表示しない（画面除外）」をそれぞれ個別のリストとして設定・管理可能とする。

### 5.3. チャンネルポイント演出機能プラグイン (Channel Points)
* 視聴者がTwitchの「チャンネルポイント」を引き換えた際、配信画面上に特定の演出を行う機能。
* **要件一覧**:
  * **カスタム報酬の自動同期**:
    * Twitch APIを介して、チャンネルに既に登録されているカスタム報酬のリストを自動取得・同期する。
  * **マルチメディア演出の構成**:
    * 報酬ごとに「音声」「画像（アニメーションGIF含む）」「動画」「文字列（リスナーの入力テキストなど）」またはそれらの組み合わせを自由に指定・再生できる。
    * 演出の表示持続時間（秒数）を個別に設定可能とする。
  * **視覚的レイアウト編集**:
    * 画面上の「表示位置（座標）」および「表示サイズ」を、ドラッグ＆ドロップ等の直感的なGUIを用いて視覚的に配置・設定できるエディタ機能を提供する。

### 5.4. フォロワー管理機能プラグイン (Followers)
* チャンネルのフォロワー情報を管理・可視化し、配信中のアクションにつなげる機能。
* **要件一覧**:
  * 新規フォロー発生時の自動検知と、チャット欄への感謝メッセージ自動投稿機能。
  * 配信中のフォロー目標値（ゴール）の設定と、進捗の表示機能。

### 5.5. コメントモデレーション機能プラグイン (Moderation)
* 配信中のスパムや不適切な発言を自動的に検知・対処するモデレート支援機能。
* **要件一覧**:
  * NGワードリストの設定。
  * NGワード検知時の自動アクション（コメントの削除、一時的なアクセス制限（タイムアウト）、または永久追放（BAN））の実行。
  * 特定のモデレーターやBotを自動モデレーションの対象から除外するホワイトリスト機能。
