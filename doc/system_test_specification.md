# TwitchChannelManagementTool 総合テスト仕様書 (ドラフト)

本ドキュメントは、要件定義に対応する「総合テスト（System Test）」の検証項目を定義するものです。本テストは開発者およびユーザーが手動で実施し、実環境（実際のTwitchサーバー、本物のOBS、稼働している棒読みちゃん/VOICEVOX）に接続して、UXおよび要求機能が完全に満たされているかを確認します。

---

## 1. 総合テスト方針
* **対象範囲**: アプリケーション全体のエンドツーエンド（E2E）動作、およびユーザーエクスペリエンス（UX）の検証。
* **テスト環境**: 
  - Windows 10/11 動作PC
  - 実際の Twitch 配信アカウント（接続テスト用）
  - 本物の OBS Studio（ブラウザソースの接続検証用）
  - ローカルで起動された 棒読みちゃん (TCPポート8080) および VOICEVOX (HTTPポート50021)
* **検証方法**: 手動によるチェックリスト確認。

---

## 2. 総合テストケース一覧

### 2.1. 画面構成および基本UI操作 (UX検証)
* **ST_UI_001: プラグインが無い状態（初期状態）およびレイアウト確認**
  - **確認手順**:
    1. プラグインDLLをすべて別フォルダに移動し（plugins内を空にする）、アプリを起動する。
    2. 左サイドバーに「設定」タブのみが表示され、そのアイコンが `pic/Config.png` になっていることを確認する。
    3. 「設定」画面が右側のコンテンツエリアに表示されていることを確認する。
  - **期待される結果**:
    - サイドバーには「設定」のみが配置され、起動時に即座に設定画面が表示されること。
* **ST_UI_002: プラグインの動的有効・無効化の確認 (再起動なし)**
  - **確認手順**:
    1. プラグイン一覧（リスト）のチェックボックスをONにし、「設定保存」を押す。
    2. サイドバーにプラグインのタブ（取得されたアイコン画像および名前）が動的に挿入されることを確認する。
    3. チェックボックスをOFFにし、「設定保存」を押す。
  - **期待される結果**:
    - 再起動することなく、即座にサイドバーから対象プラグインのタブが削除されること。
* **ST_UI_003: プラグイン表示順の並び替え確認 (上へ / 下へ ボタン)**
  - **確認手順**:
    1. 複数のプラグインを有効化し、サイドバーに並んでいることを確認する。
    2. 設定画面のリストで下側のプラグインを選択し、`[上へ]` ボタンをクリックして順序を入れ替え、「設定保存」を押す。
  - **期待される結果**:
    - 再起動なしで、サイドバーのタブ表示順が即座に入れ替わること。
    - 「設定」タブは、並び替えの有無に関わらず常に**最下部に固定**されて表示されていること。
* **ST_UI_004: デフォルトアセット自動展開および上書き制限の確認**
  - **確認手順**:
    1. プラグインを新規に有効化して保存する。
    2. `assets/overlay/[プラグイン名]/default/` フォルダが自動作成され、中にアセットファイル（index.html等）が自動抽出されていることを確認する。
    3. 展開された `index.html` をメモ帳等で開き、適当な文字（「テスト編集」等）を追記して上書き保存する。
    4. アプリを再起動、またはプラグインを一度無効化して再度有効化する。
  - **期待される結果**:
    - `index.html` がプラグインのDLL内蔵アセットで自動上書きされず、手動で追記した「テスト編集」という文字列が残っていること（ユーザーの直接編集が保護されていること）。
    - 手動で `default/` フォルダを削除して再度プラグインをロードすると、新品のデフォルトファイルが再展開されること。


### 2.2. Twitch接続およびデータ受信の確認 (E2E)
* **ST_TWC_001: Twitchログイン(OAuth)および接続テスト**
  - **確認手順**:
    1. チャンネル名とOAuthアクセストークンを入力し、「接続」ボタンを押す。
  - **期待される結果**:
    - ステータスバーの接続状態表示が「接続完了」に変化すること。
* **ST_TWC_002: コメント受信の確認**
  - **確認手順**:
    1. 対象のTwitchチャンネルのチャット欄から、別のTwitchアカウントでコメントを投稿する。
  - **期待される結果**:
    - 本ツールで読み込まれているコメント対応プラグインの画面に、投稿されたコメントが瞬時に表示されること。

### 2.3. OBS 連携機能の確認 (E2E)
* **ST_OBS_001: OBSブラウザソース接続およびコメント表示の確認**
  - **確認手順**:
    1. OBSを起動し、ソースに「ブラウザソース」を追加する。
    2. URLに `http://localhost:58081/`（または設定した待機URL）を入力する。
    3. Twitchでチャットコメントを送信する。
  - **期待される結果**:
    - OBS上のブラウザソース表示（画面）に、送信されたコメントが浮遊・バウンドして描画されること。
    - アバター画像やバッジ情報、スタンプが崩れずに描画されること。

### 2.4. 音声合成ソフト（TTS）連携の確認 (E2E)
* **ST_TTS_001: 棒読みちゃんによるコメント読み上げの確認**
  - **確認手順**:
    1. 棒読みちゃんをPC上で起動する。
    2. 本ツールで「棒読みちゃん」を選択して保存する。
    3. Twitchチャットから「テスト読み上げ」と送信する。
  - **期待される結果**:
    - 棒読みちゃんから音声が再生され、「テスト読み上げ」と読み上げられること。
* **ST_TTS_002: VOICEVOXによるコメント読み上げの確認**
  - **確認手順**:
    1. VOICEVOXをPC上で起動する.
    2. 本ツールで「VOICEVOX」を選択し、話者（ずんだもん等）を選択して保存する.
    3. Twitchチャットから「ずんだもんテスト」と送信する.
  - **期待される結果**:
    - VOICEVOXから選択した話者の声で「ずんだもんテスト」と読み上げられること.

### 2.5. Discord連携（Webhook送信）機能の確認 (E2E)
* **ST_DIS_001: プラグイン経由のDiscord Webhook送信テスト**
  - **確認手順**:
    1. 本物のDiscordサーバーでWebhookを作成し、Webhook URLをコピーする。
    2. Webhook送信に対応したプラグイン（時報プラグイン等）の設定画面を開き、Webhook URLを入力して保存する。
    3. プラグインの通知送信トリガー（テスト送信ボタンのクリックや指定イベントの発生）を発火させる。
  - **期待される結果**:
    - 該当するDiscordテキストチャンネルに、プラグインから設定された内容（メッセージや埋め込みデータ等）が正しく投稿されること。
